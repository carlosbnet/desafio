# Desafio

Aplicação para integração com a API do WhatsApp via Z-API e armazenamento de dados de clientes no Supabase.

---

## Estrutura da Tabela `Clientes`

### Colunas

| Nome | Tipo | Restrições |
|--------|----------|-------------|
| `id` | `bigint` | Chave primária (Identity) |
| `created_at` | `timestamptz` | Data de criação automática |
| `nome` | `varchar` | Obrigatório (`NOT NULL`) |
| `telefone` | `varchar` | Obrigatório (`NOT NULL`) |

### Script SQL

```sql
create table public."Clientes" (
  id bigint generated by default as identity not null,
  created_at timestamp with time zone not null default now(),
  nome character varying not null,
  telefone character varying not null,
  constraint Clientes_pkey primary key (id)
) TABLESPACE pg_default;
```

---

## Configuração do Projeto

### 1. Criar uma instância na Z-API

Acesse: https://z-api.io

Após criar sua instância, obtenha os seguintes dados:

- `ID_INSTANCIA`
- `TOKEN_INSTANCIA`

Adicione essas informações ao arquivo `.env`.

---

### 2. Configurar o Supabase

1. Crie um projeto no Supabase.
2. Execute o script SQL apresentado acima para criar a tabela `Clientes`.
3. Copie a URL do projeto e as credenciais necessárias.
4. Configure as variáveis correspondentes no arquivo `.env`.

---

## Executando a Aplicação

Após concluir as configurações, execute:

```bash
python3 main.py
```

---

## Requisitos

- Python 3.x
- Conta na Z-API
- Projeto Supabase configurado
- Arquivo `.env` configurado corretamente
